01 · The north star

The system serves a relationship, not a retrieval demo.

The nearest reference is Samantha in Her: quick, curious, warm, funny, opinionated, useful, and already caught up. Recall should feel like someone who knows the shape of your life—not software proving it found a database row.

01

Already present

It enters a session with a bounded model of current life, the recent arc, unresolved situations, forward intentions, boundaries, and shared history.

02

Genuinely curious

Questions follow real loose ends. It may react, tease, observe, help, change the channel, or stay quiet—never run a scripted interview.

03

Reliably honest

It distinguishes what the user said, what Recall inferred, what used to be true, what conflicts, and what did not surface.

04

Relationship-aware

It remembers its own promises and mistakes, repairs specifically, earns callbacks, learns delivery preferences slowly, and drops humor instantly when the room turns serious.

What it is
01
A voice-first second brain with an agenda

One dark screen, one expressive orb, memory as a living constellation, and a brain view for inspection.

02
A continuity engine

It notices what changed, what remains unfinished, what should return later, and what the user expects it to remember forward.

03
A mixed-initiative friend

It can bring up one relevant thing, but only after policy judges grounding, timing, cost, sensitivity, repetition, and user load.

What it refuses to become
×
A chatbot that dumps retrieved facts

Memory is absorbed and expressed naturally; the machinery never becomes conversational content.

×
A notification center with a personality prompt

At most one proactive aside competes with the legitimate option of intelligent silence.

×
A fake human or simulated brain

Recall borrows useful cognitive distinctions while remaining inspectable, deterministic where authority matters, and honest about what it is.

The decisive separation: retrieval answers “what could matter?” Attention answers “does it belong now?” Relationship policy answers “how should Recall carry it?” Voice performs the result. No layer is allowed to steal another layer’s authority.

04 · Memory model

Preserve what happened. Rebuild what it means.

Recall deliberately separates evidence from derived claims, current beliefs, active situations, relationship state, and prompt context. Summaries and profiles are projections, never truth.

Told · July 11“The Vienna call is July 27th.”

An immutable evidence event is stored with recorded time, source, trust, sensitivity, and a story/applicability date.

Changed · July 14“It moved to the 24th.”

A new event links to the prior telling. The write card can immediately say what this updates without silently editing history.

Projected truthJuly 24 is current.

The older interval closes; both tellings remain explainable. Equal-authority ambiguity would stay conflicting instead.

Used in conversationCurrent date leads.

Historical July 27 evidence may explain the change, but cannot become applicable truth merely because semantic search ranked it highly.

09 · The friendship layer

Recall remembers who it has been with the user.

User memory, relationship memory, and agent procedural memory remain separate. That separation is what allows humor, adaptation, trust, and repair without quietly inventing traits about the user.

User memory

Their life

People, places, projects, preferences, decisions, events, emotions, routines, commitments, and future intentions—grounded in canonical evidence.

Relationship memory

What happened between us

Recall’s promises, promise outcomes, mistakes, ruptures, explicit boundaries, repair attempts/outcomes, delivery feedback, and shared references.

Procedural memory

How Recall should change

Versioned behavior patches created only by explicit boundaries or accepted repair—not by one laugh, long session, or inferred engagement.

Rupture → repair → earned trust

01
Specific failure becomes open repair

A broken promise or concrete mistake is preserved; vague guilt is not useful state.

02
Repair outranks every proactive candidate

Humor, callbacks, anniversaries, and helpful nudges are suppressed.

03
Recall cannot accept its own apology

Only user acceptance/resolution closes the rupture.

04
Accepted repair may change behavior

One concrete procedural rule is earned; a dramatic “I’ll do better” is not.

Adaptive personality—bounded dimensions

The stable persona remains warm, quick, candid, curious, witty, and useful. The relationship can tune only delivery dimensions, never truth, safety, identity, or the attention decision.

16 · From architecture to magic

The next work is quality, not another giant subsystem.

The architecture is broad enough. The highest-value move now is deep conversational testing with real sessions, followed by focused tuning where the experience exposes gaps.

Build now
Run long-form friendship replay and real voice sessions

Test multi-session continuity, changes, unresolved threads, quiet lulls, interruptions, humor, repair, source routing, deletion, and recovery. Capture false familiarity and robotic patterns.

Build now
Tune data quality revealed by the graph

The current seeded corpus shows some over-grouped conflicting routine/relationship projections and duplicated Lila threads. Improve entity/predicate normalization from replay evidence—not by hiding UI symptoms.

Build now
Create operator replay views for attention and presence

Make “why now / why silent / why this wording” easy to inspect across complete sessions, then promote proactive classes one at a time.

Experiment
Text-derived emotional continuity in real use

Validate whether grounded episode comparison feels perceptive or invasive. Tune thresholds and language before adding a new signal.

Experiment
Voice prosody as corroborating evidence

Later, add consented per-turn valence/intensity from voice—possibly Hume or another classifier—linked as temporary evidence. It may support text; it must never diagnose or create traits.

Postpone
Multi-device sync / Postgres migration

Only when the product becomes multi-device or multi-user. Preserve event semantics, encryption, deletion, and replay before changing storage topology.

Reject
Engagement-maximizing companion behavior

No optimization for session length, dependency, constant outreach, fake inner life, or charm that bypasses safety and repair.
